# UI Generation Rules

These rules are HARD CONSTRAINTS. Every line of UI code must follow them. No exceptions unless explicitly approved by the user.

## The Test

Before writing ANY UI code, ask TWO questions:
1. "Does this look like it was designed by a human with opinions, or generated by an AI that hedged every decision?"
2. "Would a UB sophomore screenshot this and send it to their roommate?"

If either answer is no, rewrite it.

## Color Rules

1. Every color MUST come from the token list. No inline hex codes. No `text-gray-400`. No `bg-zinc-900`.
2. Backgrounds use warm blacks: `bg-void` (#0D0D0C), `bg-surface` (#121210), `bg-card` (#161614), `bg-card-hover` (#1C1C19). Pure #000 is banned — it reads as terminal, not social.
3. Text opacities: `text-white` (1.0), `text-white/70` (0.7 — secondary but important), `text-white/50` (0.5 — tertiary), `text-white/30` (0.3 — placeholders/metadata only). Four levels, no others.
4. Gold (`#FFD700`) is the brand. Use it with intent:
   - **Always gold:** Logo, active nav indicators, creation CTAs ("Deploy", "Make"), engagement numbers above threshold, the create FAB, notification badges.
   - **Gold accents:** Selected states, link text on dark surfaces, focus rings, progress indicators for creation flow.
   - **Never gold:** Body text, large background fills (>200px), every button on a page, decorative borders with no meaning.
   - **Rule of thumb:** Gold should be 15-25% of the visual weight on any screen. Not 5% (invisible brand) and not 60% (garish).
5. Status colors: `emerald-400` (success/online), `red-400` (error/live), `amber-400` (warning/pending). These have JOBS — don't use them decoratively.
6. No blue. No purple. No pink. No teal. No color that isn't in this list. The palette is warm black + white + gold + status. That's it — but USE what you have.

## Typography Rules

7. Display font: Clash Display for page titles and hero text. This is the personality font — bold, tight tracking (-0.03em), used sparingly. If the page has a title, it's Clash.
8. Body font: Geist for everything else. Regular (400) for body, Medium (500) for emphasis, Semibold (600) for card titles and buttons. No Bold (700) in Geist — that's what Clash is for.
9. Mono font: Geist Mono ONLY for labels (11px), timestamps, metadata, counters. Never for body text.
10. Size scale: page titles (32px Clash bold), section titles (20px Geist semibold), card titles (16px Geist medium), body text (15px), secondary text (14px), labels/meta (11px mono).
11. No font size below 11px. No font size above 56px (except landing hero at 64px).
12. Section labels: Geist Mono, 11px, uppercase, `text-white/50`, tracking-wider.

## Spacing Rules

13. Every spacing value MUST be a multiple of 4px. No 5px, no 7px, no 10px, no 15px.
14. Card internal padding: 16px. Card gaps: 16px. Bento grid gaps: 12px.
15. Section label to first item: 12px. Page title to first section: 24px.
16. Density contrast is intentional: tight spacing (8px gaps) in data-rich areas (vote counts, activity feeds), generous spacing (32-48px) around creation CTAs and empty states. Uniform spacing = AI sameness.

## Surface Rules

17. Cards: `bg-card` with `border border-white/[0.05]`. Rounded corners at `rounded-2xl` (16px).
18. One texture is allowed: a subtle grain overlay at 2-4% opacity on page-level backgrounds. This breaks the "CSS void" look. Use sparingly — one layer, not stacked.
19. One glow is allowed: a radial gradient from `#FFD700` at 3-5% opacity behind creation surfaces and milestone moments (first creation, response threshold hit). This signals "something is happening." Not on every surface — only on the 1-2 moments that matter per page.
20. No glass. No blur (except overlays at `rgba(0,0,0,0.85)`). No gradient fills on cards. No frosted panels. No neon.
21. No box shadows except the card hover lift: `translateY(-2px)` with `transition-transform duration-100`.

## Motion Rules

22. Default transitions: 100ms for interactions (hover, press), 200ms for state changes (phase transitions, panel reveals). Nothing longer than 300ms.
23. No decorative motion. No ambient floating. No infinite loops except the gold dot pulse.
24. Allowed motion patterns (5 total):
    - **Gold dot pulse:** 3s cycle, aliveness indicator only
    - **Card hover lift:** 100ms `translateY(-2px)`, desktop only
    - **Creation celebration:** One-shot scale-in (0→1) + gold ring ping on first creation complete. Plays once.
    - **Count-up:** Numbers animate from 0 to target over 800ms on first reveal (response counts, vote totals). Ease-out cubic.
    - **Stagger reveal:** Feed items and grid cards can stagger-enter with 50ms delay between items, 150ms fade-in each. Maximum 5 items staggered, rest appear instantly.
25. Respect `prefers-reduced-motion`: disable all of the above. Show final state immediately.
26. STILL BANNED: spring physics, bounce, elastic, magnetic, parallax, scroll-triggered transforms, `animate-pulse` on interactive elements.

## Button Rules

27. ALL buttons are pills: `rounded-full`.
28. Primary: `bg-white text-black font-semibold` (white pill).
29. Creation primary: `bg-[#FFD700] text-black font-semibold` (gold pill). ONLY for the main creation CTA ("Deploy", "Make it live", create FAB). Maximum ONE gold button visible per screen.
30. Secondary/ghost: `border border-white/10 text-white` (ghost pill)
31. Text button: `text-white/50 hover:text-white` (no background, no border)
32. No other button styles exist. No square buttons. No icon-only buttons without aria-label.

## Component Rules

33. Maximum component file: 200 lines. If longer, split it.
34. No wrapper components that only add a className.
35. No abstraction until 3+ identical usages.
36. Before creating ANY new component: grep for existing ones.
37. Before importing ANY new library: check if Tailwind + existing components can do it.
38. No default exports on components — always named exports.

## Layout Rules

39. Desktop: 200px sidebar + fluid main. No other layout.
40. Feed: max-width 640px centered in main area.
41. Space stream: full width of main area.
42. Creation: split view (prompt left, preview right) in main area.
43. Profile: max-width 960px centered, bento grid.
44. Mobile: full-width + 56px bottom bar. No sidebar.
45. One asymmetric break per surface is encouraged. Let a CTA break the content column, a header bleed edge-to-edge, or a bento tile span irregularly. Perfect symmetry = AI layout.

## Content Rules

46. User-facing copy says "apps" not "tools." Always.
47. No empty states without guidance AND personality. Tell the user what to do next in a voice that sounds like a friend, not a database.
48. No screens without outbound links to other surfaces. No dead ends.
49. No "Oops!" or cute error copy. Errors are honest and direct.
50. No placeholder text in production. No "Lorem ipsum." No "Coming soon" except the 3 defined social teases.
51. Campus-specific content in examples and previews. Use real UB names (restaurants, orgs, buildings, professors) — not generic placeholders. This is how identity works.

## Anti-Slop Patterns (NEVER DO THESE)

52. No `hover:scale-105`. Scale transforms are AI slop indicator #1.
53. No `transition-all`. Transition specific properties only.
54. No `backdrop-blur` on anything except modal overlays.
55. No `bg-gradient-to-*` on cards or surfaces. (Radial gold glow on page backgrounds is allowed per rule 19.)
56. No `shadow-lg`, `shadow-xl`, `shadow-2xl`. No heavy shadows.
57. No `animate-pulse` on buttons or cards. Only on skeleton loaders.
58. No `ring-2 ring-blue-500` focus styles. Focus ring is `outline-2 outline-[#FFD700]`.
59. No `space-y-*` nesting more than one level deep. Flatten layouts.
60. No conditional rendering trees deeper than 2 levels.
61. No `className` strings longer than ~120 chars. Extract to a variable or component.
62. No components that accept more than 8 props.
63. No `useEffect` for things that should be event handlers.
64. No `any` types. No `as unknown as`. Type it properly.
65. No purple. No purple-to-blue gradients. No teal accents. These are the statistical median of AI training data.
66. No three-card icon grid layouts. This is the canonical AI-generated layout.
67. No Inter, Roboto, or Open Sans. These are AI default fonts.

## The Smell Test

If you've written a component and it has ANY of these, rewrite it:
- More than 3 Tailwind color classes that aren't from the token list
- More than 2 animation/transition classes on a single element
- A `className` prop that overrides internal styles (breaks encapsulation)
- More than 1 level of component nesting for layout (div > div > div is max)
- A file that imports more than 5 things from the design system
- An opacity value that isn't 1.0, 0.7, 0.5, or 0.3
- Perfect visual symmetry on every element (add one intentional break)
- Zero gold on the screen (the brand should be felt)
- Pure #000000 as a background color (use warm blacks)

## When In Doubt

Write less. Use fewer classes. Use fewer props. Use fewer components. But make the one thing you DO write feel like a human chose it — not like a system generated it. One strong opinion beats ten safe defaults.
