# 0. Design Energy Zones

HIVE has **two distinct design energies** that should NOT bleed into each other:

## Zone A: Entrance (Landing â†’ Auth â†’ Onboarding)

| Property | Value |
|----------|-------|
| **Energy** | "Rap video" â€” aspirational, rule-breaking |
| **Motion** | SNAP â†’ HOLD â†’ SNAP (rhythmic, punchy) |
| **Layout** | Organic fragments, no grid |
| **Content** | Show outcomes, create FOMO |
| **Gold usage** | Live/happening (not achievement) |
| **Shell** | VoidShell, AuthShell, OnboardingLayout |

**Purpose**: Make people feel "I could be part of this"

## Zone B: Product (Feed, Spaces, Profile, HiveLab)

| Property | Value |
|----------|-------|
| **Energy** | Functional premium â€” predictable, usable |
| **Motion** | Buttery springs, 300ms default (T2) |
| **Layout** | Grid systems, max-widths, consistent spacing |
| **Content** | Information hierarchy, clear actions |
| **Gold usage** | Achievement/reward only (1% rule) |
| **Shell** | UniversalShell, ProfileShell, CanvasShell |

**Purpose**: Help people accomplish things efficiently

## Don't Cross-Contaminate

| Anti-Pattern | Why It's Wrong |
|--------------|----------------|
| SNAP motion in Feed | Product needs predictability |
| Organic fragments on Profile | Users need to find things |
| "FOMO copy" in Space settings | Functional UI needs clarity |

The landing creates an **expectation of energy**. The product delivers **tools to channel that energy**.

# 1. Shell System

Shells are the outermost layout wrappers that define the structural framework for different experiences.

**Location**: `packages/ui/src/shells/`

## Shell Selection Guide

| Shell | Use When | Max-Width | Example Pages |
|-------|----------|-----------|---------------|
| **VoidShell** | Auth, onboarding, focused flows | xs-lg (360-640px) | Login, onboarding steps |
| **UniversalShell** | Authenticated app experience | Full + sidebar | Feed, spaces, calendar |
| **StreamShell** | Vertical content flow | sm-lg (448-896px) | Feed, notifications |
| **GridShell** | Visual browse/discovery | Full with responsive grid | Space discovery, gallery |
| **ProfileShell** | Identity pages | md-xl (512-896px) | User profile, space about |
| **CanvasShell** | Three-panel creation | Full with panels | HiveLab IDE |

### Max-Width Standards

| Size | Value | Use Case |
|------|-------|----------|
| `xs` | 360px | OTP input, tight fields |
| `sm` | 420px | Login forms, single-field |
| `md` | 512px | Onboarding content |
| `lg` | 640px | Space selection, wide content |
| `xl` | 896px | Full layouts |
| `reading` | 672px | Optimal prose (~65 chars) |
| `page` | 1280px | Page container |

### Responsive Breakpoints

| Breakpoint | Width | Layout Change |
|------------|-------|---------------|
| Mobile | 0-639px | Single column, bottom nav |
| Tablet | 640-1023px | 2 columns, collapsed sidebar |
| Desktop | 1024px+ | Full layout, expanded sidebar |

### Touch Target Standards

- **Minimum**: 44x44px (Apple HIG)
- **Recommended**: 48x48px (more forgiving)
- **Spacing between**: 8px minimum

### Motion Tier Quick Reference

| Tier | Duration | Use For |
|------|----------|---------|
| **T1** | 500-700ms | Celebrations, achievements, milestones |
| **T2** | 300ms | Standard interactions, cards, filters, modals |
| **T3** | 150-200ms | Ambient, hovers, toggles, micro-feedback |
| **T4** | 0-50ms | Reduced motion fallback (accessibility) |

### Easing Functions

| Name | Value | Use |
|------|-------|-----|
| **Default** | `[0.23, 1, 0.32, 1]` | 90% of animations |
| **Snap** | `[0.25, 0.1, 0.25, 1]` | Toggles, checkboxes |
| **Dramatic** | `[0.165, 0.84, 0.44, 1]` | Achievements ONLY |
| **Premium** | `[0.22, 1, 0.36, 1]` | Apple/OpenAI feel |
