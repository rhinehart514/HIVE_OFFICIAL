Methodology

How we score

Each generation is evaluated on 5 dimensions with weighted composite scoring:

  • Intent (35%) — Right element types selected? Count-aware matching with recall-weighted F1.
  • Schema (25%) — Valid config per Zod schemas and element manifest required fields.
  • Tier (15%) — Element tiers (T1/T2/T3) appropriate for the deployment context.
  • Connections (10%) — Required data connections declared with valid instance references.
  • Functional (15%) — Config + position + size present, element exists in renderer registry.

Intent gate: If Intent scores below 50% (wrong element type entirely), the case auto-fails regardless of composite score. A well-formed wrong answer is still wrong.

Engine: Rules-based regex generator. ~100ms latency, $0 cost, no LLM. 13 element patterns, 2 smart auto-pairs (poll↔chart, RSVP↔countdown).

Suite: 60 cases across 4 levels — 20 explicit keyword, 15 multi-element, 10 implicit intent, 15 edge cases (typos, slang, emoji, negation, mixed language).
